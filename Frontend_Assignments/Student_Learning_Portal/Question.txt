Build a Student Learning Portal using React Router.
Scenario
You are asked to build a simple Student Learning Portal for an online training institute.
The portal will have public pages that anyone can view, such as Home, About, Courses, Contact, and Login.
The portal will also have private pages that only logged-in students can access, such as Dashboard, Profile, My Courses, and Settings.
The application should contain the following pages:
 
Page	Access Type	Purpose
Home	Public	Landing page of the portal
About	Public	Information about the portal
Courses	Public	Displays all available courses
Course Details	Public	Displays selected course details.
Contact	Public	Displays contact information.
Login	Public	Allows the user to log in
Dashboard	Protected	Main private student area
Profile	Protected	Displays student profile details
My Courses	Protected	Displays enrolled courses
Settings	Protected	Displays student settings
Not Found	Public	Displays 404 page for invalid URLs
 
If a user tries to access a private page without logging in, the application should redirect the user to the login page.
If a user enters a wrong URL, the application should show a proper 404 Page Not Found screen.

Mandatory Routing Concepts
The assignment must demonstrate the following concepts:
Basic Routing
The application should allow users to navigate between multiple pages without refreshing the browser.
Link
Use regular navigation links to move between pages.
NavLink
Use active navigation links in the navbar so that the current page is visually highlighted.
useNavigate
Use programmatic navigation for button clicks and actions such as login, logout, back navigation, and 404 page navigation.
useParams
Use route parameters to display selected course details based on course ID.
Navigate
Use redirection when an unauthenticated user tries to access protected pages.
Protected Route
Restrict access to private pages based on login status.
Nested Routing
Use nested routes inside the Dashboard section.
Wildcard Route
Display a 404 page when no route matches the entered URL.
User Flow Requirements
Public User Flow
A public user should be able to:
Open the Home page
View About page
View Courses page
View Course Details page
View Contact page
Open Login page
If the public user tries to access the Dashboard or any dashboard child page, the application should redirect to Login.
Login Flow
The user should:
Enter username and password
See validation messages if fields are empty
See an error message if credentials are wrong
Be redirected to Dashboard if credentials are correct
Logged-in User Flow
A logged-in user should be able to:
Access Dashboard
Access Profile
Access My Courses
Access Settings
See Dashboard and Logout options in the navbar
Logout successfully
After logout, the user should not be able to access protected pages.
Invalid URL Flow
If a user enters an invalid URL, the application should show the Not Found page.
The Not Found page should provide an option to return to the Home page.
